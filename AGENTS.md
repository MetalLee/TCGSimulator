# 项目：TCG 挂机摆摊收集游戏 MVP

你正在开发一个 Godot 4 Web 游戏，必须使用 GDScript。

## 游戏概念

这是一款轻量级 Web 端挂机卡牌收集与摆摊交易模拟游戏。

核心循环是：

1. 玩家通过挂机时间获得卡包。
2. 玩家打开卡包获得卡牌。
3. 新卡进入图鉴收藏。
4. 重复卡可以上架到自己的摊位。
5. NPC 买家和模拟玩家会购买玩家上架的卡牌。
6. 玩家获得金币。
7. 玩家用金币购买更多卡包，或者购买缺失卡牌。
8. 玩家提升图鉴完成度和摊位收益。

## 技术限制

- 使用 Godot 4.x。
- 只使用 GDScript。
- 不使用 C#。
- 目标平台是 Web。
- 第一版 MVP 先做成本地离线模拟。
- 不需要真实后端。
- 所有游戏状态使用 JSON 本地保存。
- 代码结构要方便后续把本地服务替换为 HTTP API 服务。
- UI 保持简洁、清晰、适合网页游玩。
- 优先使用容易理解和扩展的场景与脚本结构。

## MVP 必需功能

必须实现：

- 主界面 / 仪表盘
- 挂机奖励领取
- 卡包库存
- 开包
- 卡牌库存
- 图鉴
- 玩家摊位
- 上架重复卡
- NPC 买家模拟
- 模拟市场浏览
- 从模拟摊位购买卡牌
- 金币经济
- 本地保存 / 读取
- Debug 重置按钮

暂时不需要：

- 真实登录
- 真实多人
- 真实支付
- 真实后端
- 复杂动画
- 卡牌战斗效果
- 战斗系统
- 组卡系统
- 聊天
- 好友
- 排行榜
- 拍卖行

## 代码风格

- 使用清晰的类名和文件名。
- 尽量使用带类型的 GDScript。
- 避免把所有逻辑写进一个巨大脚本。
- 使用 Autoload 单例管理游戏状态和经济逻辑。
- 对重要公式和逻辑添加注释。
- 不要过度工程化。
- 每完成一个实现步骤，都要确保项目仍然可以运行。

## 验收目标

新玩家应该能够：

1. 打开游戏。
2. 领取挂机奖励。
3. 打开至少一个卡包。
4. 看到卡牌进入库存。
5. 看到图鉴进度变化。
6. 把重复卡上架到自己的摊位。
7. 模拟访客购买已上架卡牌。
8. 获得金币。
9. 用金币购买另一个卡包。
10. 保存并重新加载进度。
